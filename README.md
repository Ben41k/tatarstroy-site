# Татарстрой — структура сайта

Это статический сайт-визитка компании «Татарстрой» на нескольких HTML-страницах с общей разметкой (header/footer), стилями и клиентским JavaScript.

## Страницы

- `index.html` — главная страница (hero-блок и CTA).
- `services.html` — список услуг.
- `projects.html` — примеры проектов (карточки с изображениями из `images/`).
- `about.html` — информация о компании (история, преимущества, команда).
- `contacts.html` — контакты и форма обратной связи.

## Компоненты

Переиспользуемые фрагменты подключаются на лету через атрибут `data-include`.

- `components/header.html` — шапка и основная навигация.
- `components/footer.html` — футер и дублирующая навигация.

## Стиль

- `css/style.css` — основной файл со всеми CSS-стилями (типографика, секции, карточки, адаптивность).

## Скрипт

- `js/main.js`
  - подгружает компоненты `header.html` и `footer.html` в блоки с `data-include`;
  - подсвечивает текущий пункт меню через `aria-current="page"`;
  - валидирует форму на `contacts.html` (имя, телефон, сообщение) и показывает сообщение об успешной отправке.

## Ресурсы

- `images/` — изображения проектов (`project-1.webp` … `project-6.webp`).

## Прочее

- `file_output/` — папка под вспомогательные/генерируемые файлы (сейчас пуста).
- `.gitignore` — настройки игнора для Git.

## Запуск локально

Так как компоненты подгружаются через `fetch` из `js/main.js`, открывать страницы лучше через локальный web-сервер (а не напрямую `file://`), чтобы запросы на `components/*` работали корректно.
